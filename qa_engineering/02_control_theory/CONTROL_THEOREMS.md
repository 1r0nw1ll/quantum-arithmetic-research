## Theorem 1 — SCC Monotonicity under Generator Expansion

**Statement**: If Σ₁ ⊆ Σ₂, then #SCC(G_{Σ₂}) ≤ #SCC(G_{Σ₁}).

**Proof**: G_{Σ₂} is obtained from G_{Σ₁} by adding directed edges. Adding edges cannot split an SCC — it can only merge SCCs. □

**Engineering implication**: Adding more generators to your system can only **increase** connectivity (reduce the number of isolated components). You never lose reachability by expanding your generator set.

## Theorem 2 — SCC Count under μ-Pairing

**Statement**: Let G be the directed transition graph on Caps(N, N) generated by Σ containing μ (coordinate swap), where no other generator creates reverse paths between distinct off-diagonal states. Then the SCCs are exactly the μ-orbits: N diagonal singletons {(b,b)} and (N²-N)/2 off-diagonal pairs {(b,e), (e,b)} for b ≠ e. Thus:

```
#SCC(G) = (N² + N) / 2,    max|SCC| = 2
```

**Proof sketch**: Define V(b, e) = e. Under σ: (b,e) → (b, e+1), V increases by 1. Under λ₂: (b,e) → (2b, 2e), V increases by factor 2. Neither σ nor λ₂ can participate in a directed cycle on finite Caps. All directed cycles arise solely from μ. Each off-diagonal state (b,e) forms a 2-cycle with (e,b) under μ. Diagonal states are μ-fixed. Counting gives N + (N²-N)/2 = (N²+N)/2. □

**Engineering implication**: The **swap generator μ** is the fundamental connectivity operator. It defines the basic SCC structure. All other generators extend reachability but cannot create new cycle types beyond μ-pairing.

## Theorem 3 — Edge and Failure Counts

**Statement**: For Caps(N, N) with generators Σ ⊆ {σ, μ, λ₂, ν}, the legal transitions and failures have closed forms:

| Generator | Legal edges | Formula |
|-----------|-------------|---------|
| σ | N(N-1) | e ≤ N-1 |
| μ | N² | always legal |
| λ₂ | ⌊N/2⌋² | both coords ≤ ⌊N/2⌋ |
| ν | ⌊N/2⌋² | both coords even |

**Failures**:
- σ: N `OUT_OF_BOUNDS` failures (states with e = N)
- λ₂: N² - ⌊N/2⌋² `OUT_OF_BOUNDS` failures
- ν: N² - ⌊N/2⌋² `PARITY` failures (odd coordinates)

**Engineering implication**: You can compute the exact number of legal and illegal transitions for any Caps size before running anything. This lets you budget your search space.

**Example for N = 24**:
- σ edges: 24 × 23 = 552
- μ edges: 576 (all states)
- λ₂ edges: 12² = 144
- ν edges: 144
- σ failures: 24
- λ₂ failures: 576 - 144 = 432
- ν failures: 432

## Theorem 4 — Finite-Orbit Descent (Convergence Theorem)

**Statement**: For a quadratic loss function L on a QA orbit O of length k, with learning rate η and curvature κ_t at each step:

```
L_{t+1} = (1 - κ_t)² · L_t    [exact identity, not an approximation]
```

The **orbit contraction factor** ρ(O) = ∏(1 - κ_t)² governs convergence over a full orbit:

```
L_{t+L} = ρ(O) · L_t
```

**Key properties**:
- ρ(O) < 1 if and only if κ_min > 0 (any nonzero curvature gives convergence)
- ρ(O) is orbit-invariant — the same for all 72 starting pairs in a cosmos orbit
- Empirically: ρ = 0.001582 for mod-9, lr=0.5, gain=1 → 632× reduction per orbit

**Engineering implication**: If you can estimate the curvature κ at each QA state, you can predict the convergence rate of any process governed by QA dynamics. This is the theoretical basis for QA-guided neural network training.
