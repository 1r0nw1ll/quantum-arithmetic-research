## How It Works

### Workflow Example: Theorem Discovery

**1. Claude (Orchestrator)** designs the pipeline:
```
"We need to discover QA theorems. Here's the plan:
 - Generate balanced dataset
 - Build graph with 3 edge types
 - Train GNN for pattern learning
 - Mine conjectures via clustering
 - Verify with Lean prover"
```

**2. Codex (Code Specialist)** generates implementations:
```python
# Claude asks: "Generate graph builder"
# Codex responds with optimized code:

def build_graph_fast(tuples):
    # Efficient O(n) implementation
    lookup = {(t['b'], t['e']): i for i, t in enumerate(tuples)}
    edges = []
    for i, t in enumerate(tuples):
        if (t['b']+1, t['e']) in lookup:
            edges.append([i, lookup[(t['b']+1, t['e'])]])
    return edges
```

**3. Gemini (Analyst)** validates and provides insights:
```
"Code review:
 ✓ Correct QA tuple logic (d=b+e, a=b+2e)
 ✓ Efficient lookups (O(n) instead of O(n²))
 ⚠ Consider edge case: b+1 overflow

 Insight: The harmonic transitions form a DAG
 structure that could reveal Fibonacci sequences
 in the mod-24 residue classes."
```

**4. Claude** synthesizes:
```
"Based on Gemini's insight about DAG structure,
I'll modify the clustering algorithm to prioritize
harmonic sequence detection. Codex, implement
a custom distance metric that weights harmonic
transitions 2x more than modular edges."
```

## Advanced: Asynchronous Multi-AI

For long-running tasks, run AIs in parallel:

```python
import asyncio

async def parallel_ai_tasks():
    # Codex generates 3 implementations in parallel
    codex_tasks = [
        orchestrator.codex.invoke("Generate version A"),
        orchestrator.codex.invoke("Generate version B"),
        orchestrator.codex.invoke("Generate version C")
    ]

    # Gemini analyzes each
    results = await asyncio.gather(*codex_tasks)

    # Claude picks best
    best = claude_select_best(results)

    return best
```

## Troubleshooting

### Codex/Gemini CLI Issues

**Test connectivity:**
```bash
which codex gemini
# Should show: /usr/local/bin/codex and /usr/local/bin/gemini

codex --version
gemini --version
```

**If CLI doesn't respond:**
- Check if they're installed correctly
- Try interactive mode first: `codex` (then type prompt)
- Check API keys if required
- Review CLI documentation

### Empty Responses

The CLIs might need:
1. **Interactive mode** - They expect stdin continuously
2. **Proper prompt formatting** - Some CLIs need specific formats
3. **Authentication** - API keys might be required

**Workaround:**
Use subprocess with proper stdin handling (already implemented in `AIAgent.invoke()`)
