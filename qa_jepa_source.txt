N-JEPA objective : Given noisy QA context (\tilde{\tau}_x), predict denoised target (\tau_y) in QA space. Energy penalizes failure to “re-lock” onto the correct mod-24 phase and ellipse geometry. QA mapping: “Sparsity” = few active QA orbits : Represent a state as a mixture of QA tuples across several resonance families, but enforce that only K families / mod-24 rings are non-zero. SparseJEPA loss: Add QA-aware sparsity penalty: e.g. (\ell_1) penalty on family activations , not raw vector dims. Preserve exact QA constraints internally (no redefining J, X, K, etc.), but encourage that only a small set of family indices (resonance modes, Pisano periods) are used for any given sample. This becomes a natural concept bottleneck : each active QA family corresponds to a “concept mode” (object identity, motion style, phoneme class, etc.). ( Turing Post ) JEPA role: Predict future slices (or masked segments) of a time series from past context, purely in latent space. QA mapping: Treat time series as QA orbits : Each time step maps to a QA tuple (\tau_t). Many physical series (sensor readings, economic signals, etc.) have natural oscillations; the QA torus + ellipse geometry is ideal for capturing them. TS-JEPA objective in QA: Given context window ({\tau_{t-k}, …, \tau_t}), predictor applies QA rotor(s) to forecast (\hat \tau_{t+1:t+m}). Energy = sum of QA harmonic mismatches over the horizon: [ E = \sum_{i=1}^{m} |\Phi_\text{QA}(\hat \tau_{t+i}) - \Phi_\text{QA}(\tau_{t+i})|^2 ] Because QA supports fractional tuples , you can smoothly model slowly drifting processes while preserving discrete harmonic structure. Module: qa_jepa_timeseries.py . 3.7 TD-JEPA (Temporal-Difference JEPA) JEPA role (conceptual): JEPA that targets differences between successive latents (like TD learning in RL: predict change or cost-to-go). QA mapping: QA already has a difference variable (I = C - F) in your canonical triangle pack; and you treat deltas across time or scale as important. TD-JEPA in QA: Instead of predicting (\tau_{t+1}) directly, predict delta tuples : [ \Delta\tau_t = \tau_{t+1} - \tau_t ] subject to QA-consistent transitions (e.g. valid updates of b,e,d,a along your modular families). Energy penalizes mismatch between predicted and true (\Delta \tau_t). In RL flavor, you can let one component of the QA tuple encode cost or value (e.g. a QA-encoded scalar using your log-triangle means) and let TD-JEPA predict that component’s evolution. Simple QA rotor predictor in latent space # --------------------------------------------------------- class QARotor(nn.Module): &quot;&quot;&quot; Learnable rotor on QA tuples *via* their latent embeddings. QA-I-JEPA image model # --------------------------------------------------------- class QAImageJEPA(QAJEPAWorldModel): &quot;&quot;&quot; QA-I-JEPA: Image-based JEPA with QA world state. To do that, we: - take the latent context features from aux[&quot;h_ctx_full&quot;] - run them through QARotor - project to QA using the core qa_projector. &quot;&quot;&quot; if aux is None or &quot;h_ctx_full&quot; not in aux: # Fallback: identity prediction return context_qa h_ctx_full: torch.Tensor = aux[&quot;h_ctx_full&quot;] # [B, N, D] h_pred = self.qa_rotor(h_ctx_full) # [B, N, D] # Use the same projector as core pred_qa = self.qa_projector(h_pred) return pred_qa # ---- override forward to handle patch masking ---- def forward( self, x: torch.Tensor, y: torch.Tensor, aux: Optional[Dict] = None, ) -&gt; torch.Tensor: &quot;&quot;&quot; x: context images [B, 3, H, W] y: target images [B, 3, H, W] aux: - &quot;context_mask&quot;: [B, N] bool - &quot;target_mask&quot;: [B, N] bool For simplicity, we compute full patch grids, then apply masks inside the JEPA pipeline. In a more optimized setup, you&apos;d index patches pre- or mid-encoder like I-JEPA does. &quot;&quot;&quot; B = x.shape[0] # Encode images h_ctx_full = self.encode_context(x) # [B, N, D] h_tgt_full = self.encode_target(y) # [B, N, D] if aux is None: aux = {} aux = dict(aux) # shallow copy aux[&quot;h_ctx_full&quot;] = h_ctx_full context_mask = aux.get(&quot;context_mask&quot;, torch.ones( h_ctx_full.shape[:2], dtype=torch.bool, device=x.device )) target_mask = aux.get(&quot;target_mask&quot;, torch.ones( h_tgt_full.shape[:2], dtype=torch.bool, device=x.device )) # Select context and target subsets # Here we just keep full grid but weight loss by target_mask. # If you want actual masked subsets: index by mask before projection. # Map full grids to QA ctx_qa_full = self.qa_projector(h_ctx_full) tgt_qa_full = self.qa_projector(h_tgt_full) # Predict in QA space using rotor (which uses h_ctx_full via aux) pred_qa_full = self.predict(ctx_qa_full, aux=aux) # Compute QA energy over all patches energy_all = super().forward(x, y, aux=aux) # uses full grids internally # Optionally re-weight by target_mask (per-patch) # For now, we treat target_mask as a multiplier on the per-patch energy # by recomputing patch-wise and masking. # (This is a little redundant but explicit.) from qa_jepa_core import qa_energy, QAFeatureEmbed # avoid circular import feature_embed = QAFeatureEmbed() # patch-wise pythagorean residual res = pred_qa_full.C.pow(2) + pred_qa_full.F.pow(2) - pred_qa_full.G.pow(2) pyth = res.pow(2) # [B, N] pred_phi = feature_embed(pred_qa_full) tgt_phi = feature_embed(tgt_qa_full) feat_loss = (pred_phi - tgt_phi).pow(2).sum(-1) # [B, N] # weighted by target_mask mask = target_mask.float() pyth_weighted = (pyth * mask).sum() / (mask.sum() + 1e-8) feat_weighted = (feat_loss * mask).sum() / (mask.sum() + 1e-8) total_energy = ( self.cfg.pythagorean_weight * pyth_weighted + self.cfg.feature_weight * feat_weighted ) return total_energy 3.
